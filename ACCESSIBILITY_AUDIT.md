# Accessibility Audit for WCAG Master MVP

## Keyboard Navigation
- [ ] All buttons, inputs, and dropzones are focusable via Tab
- [ ] All actions can be triggered via keyboard (Enter/Space)

## Screen Reader
- [ ] All actionable elements have `aria-label` or descriptive text
- [ ] Error and loading states use `role="alert"` or `aria-live`

## Visual Focus
- [ ] Focus ring visible on all interactive elements

## Manual Test Results
- [ ] No blockers found in keyboard or screen reader navigation
- [ ] All error/loading states are announced
